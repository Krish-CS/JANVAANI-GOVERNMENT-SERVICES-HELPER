JANVAANI - HOW TO RUN

LIVE DEPLOYMENT URLS
- Frontend: https://janvaani-navy.vercel.app
- Backend: https://janvaani.onrender.com
- Render Python version: 3.12.0

1) Prerequisites
- Node.js 18 or newer
- Python 3.12 or newer
- Git

2) Frontend setup
- Open a terminal in the project root.
- Run: npm install
- Run: npm run dev
- Open the local Vite URL shown in the terminal.
- If you want to use the deployed frontend instead, open https://janvaani-navy.vercel.app

3) Backend setup on Windows
- Open a second terminal in the project root.
- Run: python -m venv .venv
- Run: .venv\Scripts\Activate.ps1
- Run: pip install -r backend/requirements.txt
- Copy backend/.env.example to backend/.env
- Add your API keys in backend/.env
- Run: uvicorn backend.main:app --reload --port 8000
- If you want local-only mode later, uncomment the localhost lines in backend/.env and set FRONTEND_URL back to http://localhost:5173

4) What each API key is used for
- GROQ_API_KEY: speech-to-text and primary LLM provider
- CEREBRAS_API_KEY: LLM fallback provider
- NVIDIA_API_KEY: LLM fallback provider
- AZURE_SPEECH_KEY: primary text-to-speech provider
- AZURE_SPEECH_REGION: required with Azure Speech
- SARVAM_API_KEY: fallback text-to-speech provider
- BHASHINI_USER_ID / BHASHINI_API_KEY / BHASHINI_PIPELINE_ID: fallback text-to-speech provider

5) Useful checks
- Backend health: GET https://janvaani.onrender.com/api/health
- Backend config: GET https://janvaani.onrender.com/api/config
- Local checks if you run it on your machine:
	- Backend health: GET http://localhost:8000/api/health
	- Backend config: GET http://localhost:8000/api/config

6) Build for production
- Frontend: npm run build
- Backend: deploy with uvicorn backend.main:app or the Render blueprint in backend/render.yaml
- Render uses backend/.python-version and the PYTHON_VERSION env var to stay on Python 3.12.0

7) Important notes
- Do not commit backend/.env
- The assistant is source-grounded and should rely on official context first
- If the language you speak is Hindi, Tamil, or English, the response should stay fully in that language
